# 设计规范文档（Design Specification Document）

## 1. 设计概述
- **设计理念**：以“信息降噪”为核心，打造一个清晰、直观、高效的现代甘特图工具。设计应服务于内容，通过简约的视觉语言，帮助用户快速聚焦项目核心数据，提升项目管理效率。
- **目标用户**：项目经理、产品经理、开发团队以及需要进行任务和进度管理的专业人士。
- **设计原则**：清晰直观、操作高效、一致性、可访问性。

## 2. 视觉设计规范
### 2.1 色彩系统
- **主色调 (Primary)**：`#4A90E2` (科技蓝) - 用于交互元素、选中状态和关键按钮，传达专业、高效的品牌感。
- **辅助色彩 (Accent)**：`#50E3C2` (薄荷绿) - 用于进度条、完成状态，带来积极、正向的视觉反馈。
- **中性色 (Neutral)**：
    - `#1F2937` (深灰) - 主要文字
    - `#6B7280` (中灰) - 次要文字、标签
    - `#E5E7EB` (浅灰) - 边框、分割线
    - `#F9FAFB` (背景灰) - 页面/模块背景
    - `#FFFFFF` (纯白) - 卡片背景
- **功能色彩 (Functional)**：
    - **成功**: `#22C55E`
    - **警告**: `#F59E0B` (用于表示任务即将逾期)
    - **错误**: `#EF4444` (用于表示任务已逾期)

### 2.2 字体规范
- **字体族**：Inter (英文/数字), Source Han Sans CN (中文)。这是一个现代、清晰的无衬线字体组合，在屏幕上显示效果极佳。
- **标题字体**：字重 `Semibold (600)`，用于页面标题、重要模块标题。
- **正文字体**：字重 `Regular (400)`，字号 `14px`，行高 `22px`，用于表格内容等。
- **按钮文字**：字重 `Medium (500)`，字号 `14px`。
- **辅助文字**：字重 `Regular (400)`，字号 `12px`，用于时间轴刻度、表单提示等。

### 2.3 布局系统
- **栅格系统**：内容区域采用 12 列栅格，`Gutter` 间距 `24px`。
- **间距规范**：基于 8px 的倍数原则 (`8px`, `16px`, `24px`, `32px`...)，确保元素间距的统一性和节奏感。
- **圆角规范**：
    - **按钮/输入框**: `8px`
    - **卡片/弹窗**: `12px`
- **阴影系统**：使用柔和的阴影来区分层级，避免使用纯黑。
    - **标准**: `0px 4px 6px -1px rgba(0, 0, 0, 0.1), 0px 2px 4px -1px rgba(0, 0, 0, 0.06)`
    - **悬停/聚焦**: `0px 10px 15px -3px rgba(0, 0, 0, 0.1), 0px 4px 6px -2px rgba(0, 0, 0, 0.05)`

## 3. 交互设计规范
### 3.1 导航系统
- **主导航**： 这是一个单页应用，主导航简化为页面顶部的标题栏，可放置Logo、项目名称和用户头像。
- **页面跳转**：无多页面跳转，所有操作在当前视图内通过弹窗或侧边栏完成。
- **面包屑**：不适用。

### 3.2 交互反馈
- **按钮状态**：
    - **默认**: 主色背景，白色文字。
    - **悬停**: 主色背景变亮 `10%`，添加标准阴影。
    - **点击**: 主色背景变暗 `10%`。
    - **禁用**: `E5E7EB` (浅灰) 背景，`6B7280` (中灰) 文字，无阴影。
- **表单交互**：输入框默认状态为浅灰边框，聚焦时边框变为 `4A90E2` (主色)，并带有聚焦阴影。
- **加载状态**：使用柔和的骨架屏（Skeleton Screen）或在操作区域显示一个平滑的环形加载动画。
- **错误提示**：在表单项下方以红色文字展示，或在页面顶部弹出非侵入式的提示条 (Toast)。

### 3.3 动效规范
- **过渡效果**：所有状态变化的过渡动画时长 `0.3s`，缓动函数 `ease-in-out`。
- **微交互**：
    - 任务条悬停时，显示一个 `2px` 的主色描边，并显示操作工具提示。
    - 拖拽任务条时，任务条本身有轻微的放大和半透明效果。
- **加载动画**：一个由主色构成的平滑旋转圆环。

## 4. 组件设计规范
### 4.1 基础组件
- **按钮组件**：提供主按钮、次要按钮（白底主色文字）、文本按钮三种样式。
- **输入组件**：标准的文本输入框、日期选择器、下拉选择器。
- **展示组件**：
    - **卡片 (Card)**：用于弹窗和信息展示，使用 `12px` 圆角和标准阴影。
    - **标签 (Tag)**：用于标记任务状态，使用不同功能色，`4px` 圆角。

### 4.2 业务组件
- **表格区域 (Grid)**：
    - 表头背景色为 `#F9FAFB`，文字 `Semibold`。
    - 行之间使用 `E5E7EB` 分割线。
    - 悬停行背景色变为 `#F3F4F6`。
    - 任务层级通过 `24px` 的缩进表示。
- **时间轴 (Timeline)**：
    - 周末背景色使用 `#F9FAFB` 标记。
    - 今天（Today）的位置用一条 `1px` 的主色虚线贯穿整个时间轴。
- **任务条 (Task Bar)**：
    - 圆角 `4px`。
    - 默认背景色为 `#A5C8F1` (主色的浅色版)。
    - 进度条覆盖在任务条上，使用辅助色 `#50E3C2`。
    - 逾期任务条变为错误色 `#EF4444`。

## 6. 响应式设计
- **断点设置**：桌面端 (>1024px), 平板端 (768px-1023px), 移动端 (<767px)。
- **布局适配**：
    - **桌面端**: 完整展示表格和时间轴。
    - **平板端**: 可考虑隐藏部分次要表格列，以保证时间轴有足够的显示空间。
    - **移动端**: 甘特图在小屏幕上体验不佳，建议转为只读的任务列表视图，点击列表项可查看任务详情。
- **组件适配**：按钮、输入框等组件尺寸在移动端适当增大，以优化触摸体验。

## 7. 开发交付说明
- **设计资产**：所有图标均使用 SVG 格式，确保清晰度和可维护性。
- **技术建议**：
    - 建议使用 CSS 自定义变量 (Variables) 来实现色彩系统，方便后续实现暗黑模式 (Dark Mode)。
    - `dhtmlx-gantt` 库支持通过 CSS 模板高度自定义样式，本规范旨在覆盖其默认样式。
- **实现注意事项**：请重点关注交互动效的流畅性，避免卡顿。任务条的拖拽和缩放体验是设计的核心，需要精细打磨。
