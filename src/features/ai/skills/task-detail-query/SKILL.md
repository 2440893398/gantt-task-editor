# 任务详情查询

获取单个任务的完整信息和其子任务列表。

## 可用工具

| 工具 | 用途 | 参数 |
|------|------|------|
| get_task_detail | 任务完整详情 | task_id |
| get_subtasks | 子任务列表 | task_id |

## 工作流

### 场景：用户问"这个任务的详情"

1. 调用 `get_task_detail({ task_id })`
2. 仅展示必要信息：名称、状态、进度、时间、负责人
3. 如有子任务，提示可进一步查看

### 场景：用户问"子任务列表"

1. 调用 `get_subtasks({ task_id })`
2. 列表展示子任务及状态（单行简洁）

## 输出格式

使用引用格式标注任务：`[#1.2] 任务名称`

推荐单行格式：
- `• [#1.2] 任务名 | 状态: 进行中 | 进度: 40% | 截止: 2026-01-30`

## 注意事项

- 不要编造数据
- 使用 `[#层级] 任务名` 格式引用任务
- 优先简洁输出，不展开冗长描述
- 详情让用户通过引用链接点击查看
